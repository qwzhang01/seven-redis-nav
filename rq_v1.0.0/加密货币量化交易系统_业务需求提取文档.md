## 1. 执行摘要

### 1.1 项目概述

本项目旨在构建一个**面向个人投资者和专业量化团队的加密货币量化交易平台**，提供从行情数据、策略研发、回测验证到实盘交易的全流程支持。系统对标行业领先产品（米筐、聚宽），并参考知行量化的产品形态，采用 **Python + Go 混合架构**实现高性能与研发效率的平衡。

### 1.2 核心价值主张

| 维度 | 价值描述 |
|------|----------|
| **降低门槛** | 提供可视化策略配置和丰富模板，让非编程用户也能参与量化交易 |
| **提升效率** | 毫秒级行情推送、高性能回测引擎，支持快速策略迭代 |
| **风险可控** | 完善的风控体系和模拟交易环境，保护用户资金安全 |
| **生态赋能** | 策略商城促进优质策略流通，形成社区生态 |
| **AI 赋能** | 🌟 **系统亮点**：提供 AI 驱动的策略创建能力，支持自然语言生成策略，大幅降低量化门槛 |

### 1.3 关键数据指标

- **目标用户规模**：上线6个月内达到 10,000+ 注册用户
- **策略交易量**：日均策略执行订单 50,000+ 笔
- **系统可用性**：99.9%（全年停机时间 < 8.76小时）
- **行情延迟**：< 100ms（从交易所到用户终端）

## 2. 业务背景与愿景

### 2.1 市场背景

加密货币市场具有以下特点，催生了量化交易的强烈需求：

| 特点 | 描述 | 量化交易优势 |
|------|------|--------------|
| **24/7 交易** | 全年无休，人工盯盘不现实 | 自动化执行，无需人工值守 |
| **高波动性** | 日波动率常超过10% | 程序化捕捉交易机会 |
| **多交易所** | 存在价格差异和套利空间 | 跨所套利策略执行 |
| **数据丰富** | 链上+交易数据可追溯 | 数据驱动的策略研发 |

### 2.2 竞品分析

#### 2.2.1 知行量化 (主要参考)

| 功能模块 | 知行量化特点 | 本项目定位 |
|----------|--------------|------------|
| 行情展示 | TradingView集成，专业K线 | 对标实现 |
| 手动交易 | 现货+合约，多交易所 | 对标实现 |
| 策略广场 | 订阅制，收益展示 | 对标实现，增加免费策略 |
| 策略回测 | 可视化回测报告 | 对标实现，增强指标 |
| 风控系统 | 基础风控规则 | 增强：多维度风控 |

#### 2.2.2 行业对标

| 产品 | 定位 | 借鉴点 |
|------|------|--------|
| **米筐 (RiceQuant)** | 专业量化研究平台 | 研究环境、因子库、回测框架 |
| **聚宽 (JoinQuant)** | 量化策略社区 | 策略商城模式、社区运营 |
| **派网 (Pionex)** | 网格交易机器人 | 简易配置、自动化程度 |

### 2.3 业务愿景

> **愿景声明**：成为加密货币领域最易用、最可靠的量化交易平台，让每一位投资者都能享受量化交易带来的价值。

**三年发展路线**：

```
Year 1: 产品MVP → 核心功能完善 → 1万用户
Year 2: 策略生态 → 专业版推出 → 10万用户  
Year 3: 机构服务 → API开放平台 → 50万用户
```

#### **BG-008: 🌟 多元化策略创建能力（AI赋能）** {#bg-008}

> ⭐ **系统亮点与特色** - **不在MVP范围内**，计划在 **二期（v1.5-v2.0）** 或 **三期（v2.0+）** 开发
>
> ⚠️ **重要说明**：
> - **AI策略生成功能不属于MVP范围**
> - 用户输入自然语言描述，AI自动生成可执行策略代码
> - 大模型主要使用市面主流供应商（GPT-4/Claude/通义千问等），需要购买Token
> - 目前自然语言生成代码技术已经很成熟和主流
> - **生成的代码由用户自己审核**，也可以由用户自己跑回测、模拟、实盘，具体由用户自己决定

| SMART要素 | 描述 |
|-----------|------|
| **Specific** | 提供三种策略创建方式，满足从专业开发者到零编程基础用户的全谱系需求：<br>① **代码编写**：按规范直接编写 Python 策略代码<br>② **AI 训练**：使用机器学习训练交易机器人（AI Bot）<br>③ **自然语言**：用自然语言描述策略，系统利用大模型自动生成代码或 AI 机器人 |
| **Measurable** | 自然语言策略生成成功率 ≥85%；AI 机器人训练完成时间 < 30分钟（基础版）；大模型响应时间 < 10秒 |
| **Achievable** | 对接主流大模型供应商 API（GPT-4/Claude/通义千问）；购买Token按量计费；构建策略代码生成的 Prompt 工程框架；ML 训练采用强化学习或监督学习 |
| **Relevant** | 大幅降低量化交易门槛，让零编程基础的用户也能创建个性化策略，形成差异化竞争力。**生成代码由用户自主审核和决定是否执行** |
| **Time-bound** | v1.5 上线基础自然语言策略生成；v2.0 完善 AI 训练功能；v2.5 优化多模型支持 |

**验收标准**：
- [ ] AC-008-1: 支持 Python 代码直接编写策略，提供标准模板和规范文档
- [ ] AC-008-2: 支持自然语言输入，系统生成可运行的策略代码
- [ ] AC-008-3: 支持 AI 机器人训练，用户可配置训练参数和目标
- [ ] AC-008-4: 大模型调用需要消耗 Token，Token 通过平台购买
- [ ] AC-008-5: 生成的策略代码需通过语法检查和基础安全审查

**三种策略创建方式详情**：

| 方式 | 目标用户 | 描述 | 技术实现 | 优先级 |
|------|----------|------|----------|--------|
| **① 代码编写** | 专业开发者、量化工程师 | 按照平台规范直接编写 Python 策略代码，完全自定义逻辑 | 在线 IDE + 策略 SDK + API 文档 | P0 (已有) |
| **② AI 训练** | 进阶用户、数据科学爱好者 | 使用机器学习/深度学习训练交易机器人，无需编写具体逻辑 | 强化学习（PPO/A3C）/ 监督学习 / LSTM/Transformer | P1 (v2.0) |
| **③ 自然语言** | 小白用户、非技术背景投资者 | 用自然语言描述策略意图，大模型自动生成策略代码或 AI 机器人配置 | LLM + Prompt Engineering + Code Generation | P1 (v1.5) |

**方式①：代码编写（Python）**

```
┌─────────────────────────────────────────────────────────────────┐
│                     策略代码编写流程                              │
├─────────────────────────────────────────────────────────────────┤
│  1. 选择策略模板（网格/趋势跟踪/均值回归/套利...）                 │
│  2. 在线 IDE 编辑策略代码                                        │
│  3. 语法检查 & 安全审查                                          │
│  4. 回测验证                                                     │
│  5. 部署运行                                                     │
└─────────────────────────────────────────────────────────────────┘
```

| 功能点 | 描述 | 优先级 |

| 策略模板库 | 提供 10+ 种经典策略模板 | P0 |
| 在线代码编辑器 | Monaco Editor，支持语法高亮、自动补全 | P0 |

## 9. 附录

### 9.1 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 量化交易 | Quantitative Trading | 使用数学模型和计算机程序进行交易决策的方式 |
| 策略回测 | Backtesting | 用历史数据验证策略有效性的过程 |
| 现货交易 | Spot Trading | 按当前市场价格买卖实际资产的交易 |
| 合约交易 | Futures/Perpetual Trading | 交易资产未来价格的衍生品交易 |
| 杠杆 | Leverage | 用借入资金放大投资的机制 |
| 做多 | Long | 预期价格上涨而买入的交易方向 |
| 做空 | Short | 预期价格下跌而卖出的交易方向 |
| 逐仓 | Isolated Margin | 每个仓位独立计算保证金的模式 |
| 全仓 | Cross Margin | 账户资金共享作为保证金的模式 |
| 模拟交易 | Paper Trading | 使用虚拟资金进行模拟交易的环境 |
| 夏普比率 | Sharpe Ratio | 衡量风险调整后收益的指标 |
| 最大回撤 | Maximum Drawdown | 从历史最高点到最低点的最大跌幅 |
| 🌟 大模型 | Large Language Model (LLM) | 基于深度学习的自然语言处理模型，如 GPT-4、Claude 等 |
| 🌟 强化学习 | Reinforcement Learning (RL) | 通过奖励信号学习最优策略的机器学习方法 |
| 🌟 AI Token | AI Token | 用于消费 AI 能力（策略生成、模型训练）的平台虚拟货币 |
| 🌟 Prompt 工程 | Prompt Engineering | 设计和优化大模型输入提示的技术 |
| 🌟 交易机器人 | Trading Bot | 自动执行交易策略的程序或 AI 模型 |
