## US-042: 用户注册

### 用户故事
> 作为 访客，
> 我希望 注册一个平台账户，
> 以便 使用完整的量化交易功能。

---

### 验收标准

#### AC1: 使用邮箱成功注册

**场景描述**: 访客使用有效邮箱完成注册流程

**Given** 前置条件
- 访客在注册页面
- 访客未拥有平台账户
- 邮件服务正常

**When** 用户操作
- 访客输入邮箱 "user@example.com"
- 访客点击"获取验证码"按钮
- 访客输入收到的6位验证码
- 访客设置密码 "Password123"
- 访客确认密码 "Password123"
- 访客勾选"已阅读并同意用户协议"
- 访客点击"注册"按钮

**Then** 预期结果
- 系统显示"注册成功"
- 自动登录并跳转到首页
- 用户可以使用已注册邮箱登录
- 发送欢迎邮件到注册邮箱

---

#### AC2: 使用手机号成功注册

**场景描述**: 访客使用有效手机号完成注册流程

**Given** 前置条件
- 访客在注册页面
- 短信服务正常

**When** 用户操作
- 访客选择"手机号注册"
- 访客输入手机号 "13800138000"
- 访客点击"获取验证码"按钮
- 访客输入收到的6位短信验证码
- 访客设置密码 "Password123"
- 访客勾选"已阅读并同意用户协议"
- 访客点击"注册"按钮

**Then** 预期结果
- 系统显示"注册成功"
- 自动登录并跳转到首页
- 用户可以使用已注册手机号登录

---

#### AC3: 邮箱已被注册提示

**场景描述**: 访客使用已注册的邮箱尝试注册

**Given** 前置条件
- 邮箱 "existing@example.com" 已被注册
- 访客在注册页面

**When** 用户操作
- 访客输入邮箱 "existing@example.com"
- 访客点击"获取验证码"按钮

**Then** 预期结果
- 显示错误提示："该邮箱已被注册，请直接登录或使用其他邮箱"
- 提供"去登录"链接
- 不发送验证码

---

#### AC4: 密码强度校验 - 数据驱动

**Scenario Outline**: 验证不同密码的强度校验结果

**Given** 前置条件
- 访客在注册页面
- 已获取验证码

**When** 用户操作
- 访客输入密码 <密码>
- 访客点击密码输入框外（触发校验）

**Then** 预期结果
- 系统显示 <校验结果>
- 密码强度指示器显示 <强度等级>

**Examples**:
| 密码 | 校验结果 | 强度等级 |
|------|---------|---------|
| 123456 | 密码至少8位，需包含数字和字母 | 弱 |
| password | 密码至少8位，需包含数字和字母 | 弱 |
| Pass123 | 密码至少8位 | 弱 |
| Password1 | 校验通过 | 中 |
| Password123! | 校验通过 | 强 |

---

#### AC5: 验证码过期处理

**场景描述**: 访客输入过期的验证码

**Given** 前置条件
- 访客已获取验证码
- 验证码有效期5分钟
- 已过5分钟

**When** 用户操作
- 访客输入过期的验证码
- 访客点击"注册"按钮

**Then** 预期结果
- 显示错误提示："验证码已过期，请重新获取"
- "重新获取"按钮可点击
- 原验证码输入框清空

---

### 边界条件

| 边界场景 | 输入条件 | 预期行为 |
|---------|---------|---------|
| 邮箱格式错误 | 输入 "invalid-email" | 提示"请输入正确的邮箱地址" |
| 手机号非11位 | 输入 "1380013800" | 提示"请输入11位手机号" |
| 密码两次不一致 | 密码不同 | 提示"两次输入的密码不一致" |
| 未勾选用户协议 | 不勾选协议 | 注册按钮不可点击，提示勾选 |
| 验证码连续获取 | 60秒内再次点击 | 按钮禁用，显示倒计时 |
| 验证码错误3次 | 连续3次输入错误 | 锁定10分钟，提示稍后再试 |

---

### 性能要求

| 性能指标 | 要求 | 测试方法 |
|---------|------|---------|
| 页面加载时间 | <2秒 | 页面完全加载到可交互 |
| 验证码发送时间 | <10秒 | 从点击到收到验证码 |
| 注册响应时间 | <3秒 | 从点击注册到显示结果 |
| 并发注册 | 100并发 | 压测100个用户同时注册 |

---

### 界面要求

**必要元素**:
- 邮箱/手机号切换标签
- 邮箱/手机号输入框
- 验证码输入框 + 获取按钮
- 密码输入框（带显示/隐藏切换）
- 确认密码输入框
- 用户协议勾选框 + 协议链接
- 注册按钮
- 已有账号？去登录 链接

**交互行为**:
- 输入框失焦时实时校验
- 密码强度实时显示
- 验证码获取后60秒倒计时
- 注册中按钮显示加载状态

---

### 测试建议

**测试类型**: 功能测试 + 安全测试 + 性能测试
**测试数据**: 
- 有效邮箱和手机号
- 各种格式错误的输入
- 已注册的账号
**自动化建议**: 高优先级，核心注册流程适合E2E自动化

## US-043: 用户登录

### 用户故事
> 作为 注册用户，
> 我希望 登录我的账户，
> 以便 访问个人数据和交易功能。

---

### 验收标准

#### AC1: 使用邮箱密码成功登录

**场景描述**: 注册用户使用正确的邮箱和密码登录

**Given** 前置条件
- 用户已注册，邮箱为 "user@example.com"
- 密码为 "Password123"
- 用户在登录页面

**When** 用户操作
- 用户输入邮箱 "user@example.com"
- 用户输入密码 "Password123"
- 用户点击"登录"按钮

**Then** 预期结果
- 系统显示"登录成功"
- 跳转到首页/用户之前访问的页面
- 页面显示用户昵称/头像
- 登录状态保持7天

---

#### AC2: 使用验证码登录

**场景描述**: 用户使用手机号验证码登录

**Given** 前置条件
- 用户已注册，手机号为 "13800138000"
- 用户在登录页面

**When** 用户操作
- 用户选择"验证码登录"
- 用户输入手机号 "13800138000"
- 用户点击"获取验证码"
- 用户输入收到的验证码
- 用户点击"登录"按钮

**Then** 预期结果
- 系统显示"登录成功"
- 跳转到首页
- 登录状态保持7天

---

#### AC3: 密码错误提示

**场景描述**: 用户输入错误密码

**Given** 前置条件
- 用户已注册，邮箱为 "user@example.com"
- 正确密码为 "Password123"

**When** 用户操作
- 用户输入邮箱 "user@example.com"
- 用户输入密码 "WrongPassword"
- 用户点击"登录"按钮

**Then** 预期结果
- 显示错误提示："邮箱或密码错误"
- 密码输入框清空
- 用户可重新输入

---

#### AC4: 连续密码错误锁定

**场景描述**: 用户连续5次输入错误密码

**Given** 前置条件
- 用户已注册
- 用户已连续4次输入错误密码

**When** 用户操作
- 用户第5次输入错误密码
- 用户点击"登录"按钮

**Then** 预期结果
- 显示错误提示："密码错误次数过多，账户已锁定15分钟"
- 登录按钮禁用15分钟
- 建议使用验证码登录或找回密码
- 15分钟后自动解锁

---

#### AC5: 记住登录状态

**场景描述**: 用户勾选"记住我"后登录

**Given** 前置条件
- 用户已注册
- 用户在登录页面

**When** 用户操作
- 用户输入正确的邮箱和密码
- 用户勾选"记住我"
- 用户点击"登录"按钮
- 登录成功后关闭浏览器
- 7天内重新打开网站

**Then** 预期结果
- 用户仍处于登录状态
- 无需重新输入密码

---

### 边界条件

| 边界场景 | 输入条件 | 预期行为 |
|---------|---------|---------|
| 账号不存在 | 输入未注册邮箱 | 提示"邮箱或密码错误"（不泄露账号存在性） |
| 邮箱大小写 | 输入 "User@Example.COM" | 正常登录（忽略大小写） |
| 前后有空格 | 输入 " user@example.com " | 自动去除空格，正常登录 |
| 登录态过期 | 7天后访问 | 跳转到登录页 |
| 异地登录 | 新设备/新IP登录 | 发送安全提醒邮件 |

## US-023: 添加交易所API配置

### 用户故事
> 作为 普通投资者，
> 我希望 添加交易所的API Key，
> 以便 系统能够代我执行交易操作。

---

### 验收标准

#### AC1: 成功添加Binance API配置

**场景描述**: 用户添加有效的Binance交易所API

**Given** 前置条件
- 用户已登录
- 用户拥有有效的Binance API Key和Secret Key
- 用户在交易所管理页面

**When** 用户操作
- 用户点击"添加交易所"按钮
- 用户选择交易所类型"Binance"
- 用户输入API Key
- 用户输入Secret Key
- 用户输入API备注名称"我的主账户"
- 用户点击"保存"按钮

**Then** 预期结果
- 系统验证API有效性
- 显示"添加成功"提示
- 交易所列表显示新添加的Binance配置
- 显示API权限信息（只读/交易）
- API Key使用AES-256加密存储

---

#### AC2: API Key格式校验失败

**场景描述**: 用户输入格式错误的API Key

**Given** 前置条件
- 用户在添加交易所页面
- 已选择交易所类型

**When** 用户操作
- 用户输入无效格式的API Key（如包含空格）
- 用户点击"保存"按钮

**Then** 预期结果
- 显示错误提示："API Key格式不正确"
- 不发起验证请求
- API Key输入框标红

---

#### AC3: API权限不足提示

**场景描述**: 用户添加的API不具备交易权限

**Given** 前置条件
- 用户在添加交易所页面
- 用户的API Key仅有只读权限

**When** 用户操作
- 用户输入只读权限的API Key
- 用户点击"保存"按钮
- 系统验证API权限

**Then** 预期结果
- 显示警告提示："该API仅有只读权限，无法执行交易操作"
- 提供选项：仍然添加（仅用于查看）或返回修改
- 用户选择添加后，该API标记为"只读"

---

#### AC4: 禁止添加提现权限API

**场景描述**: 用户尝试添加具有提现权限的API

**Given** 前置条件
- 用户在添加交易所页面
- 用户的API Key具有提现权限

**When** 用户操作
- 用户输入具有提现权限的API Key
- 用户点击"保存"按钮

**Then** 预期结果
- 显示错误提示："为保障资金安全，请使用不含提现权限的API"
- 显示如何在交易所修改API权限的指引
- 不保存该API配置

---

#### AC5: 同一交易所API数量限制

**场景描述**: 用户已添加3个Binance API，尝试添加第4个

**Given** 前置条件
- 用户已添加3个Binance API配置
- 单交易所最多3个API

**When** 用户操作
- 用户尝试添加第4个Binance API

**Then** 预期结果
- 显示提示："每个交易所最多添加3个API，请删除已有API后再添加"
- 保存按钮不可点击

---

### 边界条件

| 边界场景 | 输入条件 | 预期行为 |
|---------|---------|---------|
| API备注名称为空 | 不输入备注 | 提示"请输入API备注名称" |
| 备注名称过长 | 超过50个字符 | 限制最多输入50个字符 |
| 重复的API Key | 输入已添加的API Key | 提示"该API Key已添加" |
| Secret Key复制不完整 | Secret Key截断 | 验证失败，提示"API验证失败，请检查Secret Key" |
| 网络超时 | 验证请求超时 | 提示"验证超时，请稍后重试" |

---

### 性能要求

| 性能指标 | 要求 | 测试方法 |
|---------|------|---------|
| API验证时间 | <10秒 | 从点击保存到验证完成 |
| 加密存储时间 | <500ms | API Key加密存储耗时 |
| 页面加载时间 | <2秒 | 交易所管理页面加载 |

---

### 界面要求

**必要元素**:
- 交易所类型选择器（Binance/OKX/Bybit）
- API Key输入框
- Secret Key输入框（默认隐藏，点击显示）
- API备注名称输入框
- API权限说明文字
- 如何获取API Key的指引链接
- 保存按钮

**交互行为**:
- 添加向导分步骤展示
- Secret Key默认显示为"****"
- 保存时显示加载动画
- 验证成功显示绿色勾选

---

### 测试建议

**测试类型**: 功能测试 + 安全测试 + 接口测试
**测试数据**: 
- 有效的测试API（只读/交易权限）
- 无效格式的API
**自动化建议**: 高优先级，API配置流程适合自动化

## US-024: 验证交易所API连接

### 用户故事
> 作为 普通投资者，
> 我希望 验证API Key是否配置正确且有效，
> 以便 确保后续交易能正常执行。

---

### 验收标准

#### AC1: API连接验证成功

**场景描述**: 用户验证已添加的API配置

**Given** 前置条件
- 用户已添加有效的交易所API
- API具有只读和交易权限
- 用户在交易所管理页面

**When** 用户操作
- 用户点击API配置旁的"验证"按钮

**Then** 预期结果
- 显示验证中加载动画
- 验证成功后显示绿色"连接正常"状态
- 显示API权限详情（只读✅/交易✅/提现❌）
- 显示验证时间

---

#### AC2: API连接验证失败

**场景描述**: API Key已失效或被删除

**Given** 前置条件
- 用户已添加的API Key在交易所端已被删除

**When** 用户操作
- 用户点击"验证"按钮

**Then** 预期结果
- 显示红色"连接失败"状态
- 显示失败原因："API Key无效或已被删除"
- 提供"删除"和"更新"操作按钮

---

#### AC3: API验证超时

**场景描述**: 验证请求超过10秒无响应

**Given** 前置条件
- 交易所API服务响应慢

**When** 用户操作
- 用户点击"验证"按钮
- 请求超过10秒

**Then** 预期结果
- 显示黄色"验证超时"状态
- 提示"验证超时，请稍后重试"
- 提供"重试"按钮

---

### 边界条件

| 边界场景 | 输入条件 | 预期行为 |
|---------|---------|---------|
| IP白名单限制 | API设置了IP限制 | 显示"IP不在白名单，请在交易所添加服务器IP" |
| API权限变更 | 权限被修改 | 更新显示的权限信息 |
| 频繁验证 | 10秒内多次点击 | 按钮禁用，防止频繁请求 |

---

### 性能要求

| 性能指标 | 要求 | 测试方法 |
|---------|------|---------|
| 验证响应时间 | <10秒 | 从点击到显示结果 |
| 超时阈值 | 10秒 | 超时自动终止 |

**When** 系统触发
- 回测执行完成

**Then** 预期结果
- 回测结果与手动计算一致（误差<0.01%）
- 交易记录正确记录每笔交易
- 盈亏计算考虑手续费

---

#### AC3: 策略代码执行错误

**场景描述**: 策略代码运行时出错

**Given** 前置条件
- 策略代码包含运行时错误

**When** 用户操作
- 用户启动回测

**Then** 预期结果
- 回测停止
- 显示错误信息和错误行号
- 提供"查看代码"链接定位错误

---

### 边界条件

| 边界场景 | 输入条件 | 预期行为 |
|---------|---------|---------|
| 数据量极大 | 100万条K线 | 正常完成，可能需较长时间 |
| 策略无交易 | 无触发信号 | 显示"无交易记录" |
| 策略亏损100% | 本金亏完 | 显示"爆仓"警告 |

---

### 性能要求

| 性能指标 | 要求 | 测试方法 |
|---------|------|---------|
| 回测速度 | ≥10,000 K线/秒 | 大数据量回测 |
| 最大数据量 | 100万条K线 | 压力测试 |
| 结果生成 | 回测完成后<3秒 | 报告生成时间 |

---

### 测试建议

**测试类型**: 功能测试 + 性能测试 + 准确性测试
**测试数据**: 不同复杂度的策略代码
**自动化建议**: 高优先级

## US-039a: 查看回测核心指标

### 用户故事
> 作为 策略开发者，

### 验收标准

#### AC1: 查看收益率指标

**场景描述**: 用户查看回测收益相关指标

**Given** 前置条件
- 回测已完成
- 有交易记录

**When** 用户操作
- 用户查看回测结果页面

**Then** 预期结果
- 显示总收益率
- 显示年化收益率
- 显示累计收益金额
- 数值保留2位小数

---

#### AC2: 查看风险指标

**场景描述**: 用户查看风险相关指标

**Given** 前置条件
- 回测已完成

**When** 用户操作
- 用户查看风险指标区域

**Then** 预期结果
- 显示最大回撤（金额和百分比）
- 显示夏普比率
- 显示索提诺比率
- 显示波动率

---

#### AC3: 查看交易统计

**场景描述**: 用户查看交易统计数据

**Given** 前置条件
- 回测已完成
- 有多笔交易记录

**When** 用户操作
- 用户查看交易统计区域

**Then** 预期结果
- 显示总交易次数
- 显示胜率（盈利次数/总次数）
- 显示盈亏比（平均盈利/平均亏损）
- 显示平均持仓时间

---

#### AC4: 指标计算准确性

**场景描述**: 验证指标计算公式正确

**Given** 前置条件
- 回测已完成
- 已知预期结果

**When** 系统触发
- 指标计算完成

**Then** 预期结果
- 夏普比率计算：(年化收益率 - 无风险利率) / 年化波动率
- 最大回撤计算：(峰值 - 谷值) / 峰值
- 指标与行业标准公式一致
