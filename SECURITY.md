# 安全策略

## 支持的版本

当前正在接受安全更新的版本：

| 版本 | 是否支持 |
| ------- | -------- |
| ≥ 0.1.x | ✅ 支持 |
| < 0.1.0 | ❌ 不支持 |

## 报告安全漏洞

如果你发现了安全漏洞，**请勿通过公开的 GitHub Issue 提交**。

请通过以下方式私下报告：

1. **GitHub Security Advisories**（推荐）：[提交安全建议](https://github.com/qwzhang01/seven-redis-nav/security/advisories/new)
2. **GitHub Issues**：标记为 `security` 标签的 Issue（仅限非敏感信息）

### 报告内容

请包含以下信息：

- **漏洞描述**：清晰描述漏洞类型和影响
- **复现步骤**：如何触发该漏洞
- **影响范围**：哪些版本和平台受影响
- **修复建议**：如有请附上

### 响应流程

1. 收到报告后，我们将在 **48 小时内**确认收到
2. 在 **7 个工作日内**进行初步评估
3. 确认漏洞后，我们将：
   - 在修复期间对漏洞信息保密
   - 尽快发布修复版本
   - 在 Release Notes 中致谢报告者

## 安全相关功能说明

Seven Redis Nav 涉及以下安全敏感功能：

- **密码存储**：连接密码通过 macOS Keychain 安全存储，不会明文保存到磁盘
- **SSH 隧道**：支持密码和私钥认证，私钥密码不会持久化
- **TLS 连接**：支持配置 CA 证书、客户端证书和最低 TLS 版本
- **配置导出**：导出的配置文件不包含密码

感谢你帮助保护 Seven Redis Nav 的安全！🔒
