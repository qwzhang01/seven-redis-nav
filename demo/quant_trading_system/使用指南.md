## 系统简介

这是一个**生产级量化交易系统**，核心功能包括：

- 技术指标计算（SMA、EMA、MACD、RSI、布林带、ATR、KDJ 等）
- 策略回测引擎（支持多品种、多周期、回测进度日志、完整绩效分析）
- 实盘/模拟交易执行（Paper / Live 模式）
- AI 智能交易（基于 DQN 强化学习）
- RESTful API 服务（FastAPI，含 Swagger 文档）
- 完整的基础设施支持（TimescaleDB、Redis、Kafka、MinIO、Prometheus、Grafana）

## 安装步骤

### 1. 检查 Python 版本

```bash
python --version
# 要求：Python 3.11 或更高版本
```

### 2. 进入项目目录

```bash
cd quant_trading_system
```

### 3. 安装依赖

**方式一：使用 pip**

```bash
# 基础安装
pip install -e .

# 包含 AI 功能（强化学习、PyTorch）
pip install -e ".[ai]"

# 开发环境（含测试、格式化工具）
pip install -e ".[dev]"
```

**方式二：使用 uv（更快）**

```bash
uv sync
```

### 4. 验证安装

```bash
quant --help
```

看到命令帮助信息说明安装成功。

## CLI 命令参考

所有命令通过 `quant` 入口调用，也可以用 `uv run quant` 代替。

### `quant serve` — 启动 API 服务

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--host` | 监听地址 | `0.0.0.0` |
| `--port` | 监听端口 | `8000` |
| `--reload` | 开发模式（文件变更自动重载） | `false` |
| `--workers` | 工作进程数 | `1` |

### `quant backtest` — 运行回测

| 参数 | 说明 | 默认值 | 必填 |
|------|------|--------|------|
| `--strategy` | 策略名称 | — | 是 |
| `--symbol` | 交易对（如 `BTCUSDT`） | — | 是 |
| `--start` | 回测开始时间 | — | 是 |
| `--end` | 回测结束时间 | — | 是 |
| `--timeframe` | K线周期 | `1m` | 否 |
| `--capital` | 初始资金 | `1000000.0` | 否 |
| `--commission` | 手续费率 | `0.0004` | 否 |
| `--slippage` | 滑点率 | `0.0001` | 否 |
| `--mock` | 使用模拟数据 | `false` | 否 |

支持的时间周期：`1m` / `5m` / `15m` / `30m` / `1h` / `4h` / `1d` / `1w`

系统会自动检测策略是否为多周期策略（如 `mtf_ma_breakout`），若是则自动获取多个时间周期的数据。

### `quant trade` — 启动实盘/模拟交易

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--strategy` | 策略名称（必填） | — |
| `--symbol` | 交易对，可多次指定（必填） | — |
| `--exchange` | 交易所 | `binance` |
| `--market-type` | 市场类型 | `spot` |
| `--capital` | 初始资金（paper 模式） | `100000.0` |
| `--api-key` | 交易所 API Key | 环境变量 `EXCHANGE_API_KEY` |
| `--api-secret` | 交易所 API Secret | 环境变量 `EXCHANGE_API_SECRET` |
| `--mode` | 运行模式 `paper` / `live` | `paper` |

### `quant list-strategies` — 列出可用策略

```bash
quant list-strategies
```

### `quant list-indicators` — 列出可用指标

```bash
quant list-indicators
```

### `quant check-config` — 检查配置

```bash
quant check-config
```

## 内置策略一览

| 注册名 | 类名 | 说明 | 默认周期 |
|--------|------|------|----------|
| `ma_cross` | `DualMAStrategy` | 双均线交叉策略 | M15 |
| `macd_cross` | `MACDStrategy` | MACD 交叉策略 | H1 |
| `rsi_ob_os` | `RSIStrategy` | RSI 超买超卖策略 | H4 |
| `bollinger_band` | `BollingerBandStrategy` | 布林带策略 | H1 |
| `mtf_ma_breakout` | `MultiTimeframeMABreakoutStrategy` | 多周期趋势 + MA 回调突破策略 | M15 + H1 |

### 策略参数说明

**ma_cross（双均线交叉）**
- `fast_period=5` — 快速均线周期
- `slow_period=20` — 慢速均线周期

**macd_cross（MACD 交叉）**
- `fast_period=12`, `slow_period=26`, `signal_period=9`

**rsi_ob_os（RSI 超买超卖）**
- `period=14`, `overbought=70.0`, `oversold=30.0`

**bollinger_band（布林带）**
- `period=20`, `std_dev=2.0`

**mtf_ma_breakout（多周期 MA 突破）**
- `ma_period=11` — MA 均线周期
- `risk_reward_ratio=2.0` — 盈亏比
- `min_pullback_bars=3` — 最小回调K线数
- `max_pullback_bars=5` — 最大回调K线数

## 服务器部署

本节详细介绍如何将量化交易系统部署到 Linux 服务器上运行。

### 前置条件

- 一台 Linux 服务器（推荐 Ubuntu 20.04+ / CentOS 7+）
- 服务器已安装 Docker（>= 20.10）和 Docker Compose（>= 2.0）
- 服务器已安装 Git
- 服务器至少 2GB 内存（推荐 4GB+）
- 开放端口：8000（API）、3000（Grafana）、9090（Prometheus）

### 方式一：Docker Compose 一键部署（推荐）

这是最简单的方式，一条命令启动全部服务（包括数据库、缓存、消息队列、监控）。

#### 第一步：上传代码到服务器

```bash
# 在服务器上克隆代码
git clone <你的仓库地址> /opt/quant_trading_system
cd /opt/quant_trading_system
```

或者从本地上传：

```bash
# 在本地机器执行
rsync -avz --exclude='__pycache__' --exclude='.git' --exclude='*.pyc' \
  ./ user@your-server-ip:/opt/quant_trading_system/
```

#### 第二步：配置环境变量

```bash
cd /opt/quant_trading_system

# 编辑 .env 文件，填入生产配置
vi .env
```

生产环境关键配置：

```env
# 应用配置
APP_ENV=production
DEBUG=false
LOG_LEVEL=INFO

# 数据库（使用 docker-compose 内部地址）
DB_TIMESCALE_HOST=timescaledb
DB_TIMESCALE_PORT=5432
DB_TIMESCALE_USER=quant
DB_TIMESCALE_PASSWORD=<设置一个强密码>
DB_TIMESCALE_DATABASE=quant_trading

# Redis
REDIS_HOST=redis
REDIS_PORT=6379

# Kafka
KAFKA_BOOTSTRAP_SERVERS=kafka:9092

# 交易所 API（如需实盘交易）
BINANCE_API_KEY=<你的API Key>
BINANCE_SECRET_KEY=<你的Secret Key>
BINANCE_TESTNET=false

# 风控
MAX_POSITION_SIZE=0.1
MAX_DRAWDOWN=0.2
STOP_LOSS_PCT=0.02

# API 安全
API_JWT_SECRET_KEY=<生成一个随机密钥>
```

生成随机密钥：

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

#### 第三步：构建并启动所有服务

```bash
# 构建镜像并后台启动全部服务
docker compose up -d --build
```

首次启动会拉取镜像并构建，大约需要 5-10 分钟。

#### 第四步：检查服务状态

```bash
# 查看所有容器状态
docker compose ps

# 查看 API 服务日志
docker compose logs -f quant-api

# 健康检查
curl http://localhost:8000/health
```

所有容器状态应为 `Up`，健康检查返回类似：

```json
{"status": "healthy", "version": "1.0.0", "trading_mode": "paper"}
```

#### 第五步：验证各组件

| 组件 | 访问地址 | 说明 |
|------|----------|------|
| API 服务 | `http://<服务器IP>:8000` | 量化交易 API |
| API 文档 | `http://<服务器IP>:8000/docs` | Swagger 交互文档 |
| Grafana | `http://<服务器IP>:3000` | 监控面板（默认账号 admin/admin） |
| Prometheus | `http://<服务器IP>:9090` | 指标查询 |
| MinIO | `http://<服务器IP>:9001` | 对象存储控制台（minioadmin/minioadmin） |

### 方式二：直接部署（不使用 Docker）

适合轻量级部署或只需要回测功能的场景。

#### 第一步：安装系统依赖

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install -y python3.11 python3.11-venv python3-pip build-essential curl

# CentOS / RHEL
sudo yum install -y python3.11 python3-pip gcc make curl
```

#### 第二步：创建虚拟环境并安装

```bash
cd /opt/quant_trading_system

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -e .

# 如需 AI 功能
pip install -e ".[ai]"
```

#### 第三步：配置环境变量

```bash
# 编辑 .env 文件
vi .env
# 填入数据库、Redis 等外部服务连接信息
```

#### 第四步：启动 API 服务

```bash
source .venv/bin/activate

# 前台运行（测试用）
quant serve --host 0.0.0.0 --port 8000

# 后台运行（生产用，使用 nohup）
nohup quant serve --host 0.0.0.0 --port 8000 --workers 4 > /var/log/quant-api.log 2>&1 &
```

#### 第五步：使用 systemd 管理服务（推荐）

创建 systemd 服务文件：

```bash
sudo vi /etc/systemd/system/quant-trading.service
```

写入以下内容：

```ini
[Unit]
Description=Quant Trading System API
After=network.target

[Service]
Type=simple
User=quant
Group=quant
WorkingDirectory=/opt/quant_trading_system
Environment="PATH=/opt/quant_trading_system/.venv/bin:/usr/bin"
EnvironmentFile=/opt/quant_trading_system/.env
ExecStart=/opt/quant_trading_system/.venv/bin/quant serve --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启用并启动服务：

```bash
# 创建专用用户
sudo useradd -r -s /bin/false quant
sudo chown -R quant:quant /opt/quant_trading_system

# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable quant-trading
sudo systemctl start quant-trading

# 查看状态
sudo systemctl status quant-trading

# 查看日志
sudo journalctl -u quant-trading -f
```

### 方式三：使用 Nginx 反向代理（生产推荐）

在方式一或方式二的基础上，使用 Nginx 反向代理以支持 HTTPS 和域名访问。

#### 安装 Nginx

```bash
sudo apt install -y nginx
```

#### 配置 Nginx

```bash
sudo vi /etc/nginx/sites-available/quant-trading
```

写入：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # HTTP 重定向到 HTTPS（如配置了 SSL）
    # return 301 https://$host$request_uri;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 300s;
    }

    # Grafana 面板
    location /grafana/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/quant-trading /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 配置 HTTPS（可选，使用 Let's Encrypt）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 常见问题

### Q1: 安装时报错 "ta-lib not found"

```bash
# macOS
brew install ta-lib
pip install ta-lib

# Ubuntu / Debian
sudo apt-get install -y ta-lib
pip install ta-lib

# CentOS
sudo yum install -y ta-lib ta-lib-devel
pip install ta-lib
```

### Q2: 运行 demo.py 报错

检查步骤：
1. 确认 Python 版本 >= 3.11：`python --version`
2. 确认已安装依赖：`pip install -e .`
3. 确认在项目根目录运行：`python examples/demo.py`

### Q3: 币安 API 连接失败

- 检查网络连接（部分地区可能需要代理）
- 确认 `.env` 中的 API Key 是否正确
- 使用 `--mock` 参数先用模拟数据测试

### Q4: Docker 容器启动失败

```bash
# 查看失败原因
docker compose logs <服务名>

# 检查端口冲突
sudo lsof -i :8000
sudo lsof -i :5432

# 检查磁盘空间
df -h
```

```python
    low=df['low'].values,
    close=df['close'].values,
    volume=df['volume'].values,
)
```

### Q6: 回测结果不理想怎么办？

1. 调整策略参数（均线周期、超买超卖阈值等）
2. 使用更长时间范围的数据回测
3. 尝试不同策略（如 `mtf_ma_breakout` 多周期策略）
4. 调低手续费率和滑点率查看参数敏感性
5. 开发自定义策略（参考 `src/quant_trading_system/strategies/examples.py`）
