## 系统简介

这是一个**生产级量化交易系统**，核心功能包括：

- 技术指标计算（SMA、EMA、MACD、RSI、布林带、ATR、KDJ 等）
- 策略回测引擎（支持多品种、多周期、回测进度日志、完整绩效分析）
- 实盘/模拟交易执行（Paper / Live 模式）
- AI 智能交易（基于 DQN 强化学习）
- RESTful API 服务（FastAPI，含 Swagger 文档）
- WebSocket 实时推送（行情、交易事件、策略信号）
- 信号广场（策略信号公开展示、订阅、跟单）
- 排行榜（策略收益排行、信号准确率排行）
- 完整的基础设施支持（TimescaleDB、Redis、Kafka、MinIO、Prometheus、Grafana）

## 安装步骤

### 1. 检查 Python 版本

```bash
python --version
# 要求：Python 3.11 或更高版本
```

### 2. 进入项目目录

```bash
cd quant_trading_system
```

### 3. 安装依赖

**方式一：使用 pip**

```bash
# 基础安装
pip install -e .

# 包含 AI 功能（强化学习、PyTorch）
pip install -e ".[ai]"

# 开发环境（含测试、格式化工具）
pip install -e ".[dev]"
```

**方式二：使用 uv（更快）**

```bash
uv sync
```

### 4. 验证安装

```bash
quant --help
```

看到命令帮助信息说明安装成功。

## 快速开始

### 运行示例程序（最简单）

```bash
python examples/demo.py
```

该示例会自动生成模拟数据、计算技术指标、运行双均线策略回测并展示结果。

### 运行策略回测

```bash
# 使用模拟数据
quant backtest \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000 \
  --mock

# 使用币安真实数据（需网络连接）
quant backtest \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000
```

### 启动 API 服务

```bash
quant serve
# 访问 http://localhost:8000/docs 查看 API 文档
```

服务启动时会自动加载所有已注册策略（`stopped` 状态），策略不会自动运行，需通过 API 显式创建并启动。

### 启动模拟交易

```bash
quant trade \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --mode paper \
  --capital 100000
```

## CLI 命令参考

所有命令通过 `quant` 入口调用，也可以用 `uv run quant` 代替。

### `quant serve` — 启动 API 服务

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--host` | 监听地址 | `0.0.0.0` |
| `--port` | 监听端口 | `8000` |
| `--reload` | 开发模式（文件变更自动重载） | `false` |
| `--workers` | 工作进程数 | `1` |

### `quant backtest` — 运行回测

| 参数 | 说明 | 默认值 | 必填 |
|------|------|--------|------|
| `--strategy` | 策略名称 | — | 是 |
| `--symbol` | 交易对（如 `BTCUSDT`） | — | 是 |
| `--start` | 回测开始时间 | — | 是 |
| `--end` | 回测结束时间 | — | 是 |
| `--timeframe` | K线周期 | `1m` | 否 |
| `--capital` | 初始资金 | `1000000.0` | 否 |
| `--commission` | 手续费率 | `0.0004` | 否 |
| `--slippage` | 滑点率 | `0.0001` | 否 |
| `--mock` | 使用模拟数据 | `false` | 否 |

支持的时间周期：`1m` / `5m` / `15m` / `30m` / `1h` / `4h` / `1d` / `1w`

系统会自动检测策略是否为多周期策略（如 `mtf_ma_breakout`），若是则自动获取多个时间周期的数据。

### `quant trade` — 启动实盘/模拟交易

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--strategy` | 策略名称（必填） | — |
| `--symbol` | 交易对，可多次指定（必填） | — |
| `--exchange` | 交易所 | `binance` |
| `--market-type` | 市场类型 | `spot` |
| `--capital` | 初始资金（paper 模式） | `100000.0` |
| `--api-key` | 交易所 API Key | 环境变量 `EXCHANGE_API_KEY` |
| `--api-secret` | 交易所 API Secret | 环境变量 `EXCHANGE_API_SECRET` |
| `--mode` | 运行模式 `paper` / `live` | `paper` |

## 内置策略一览

| 注册名 | 类名 | 说明 | 默认周期 |
|--------|------|------|----------|
| `ma_cross` | `DualMAStrategy` | 双均线交叉策略 | M15 |
| `macd_cross` | `MACDStrategy` | MACD 交叉策略 | H1 |
| `rsi_ob_os` | `RSIStrategy` | RSI 超买超卖策略 | H4 |
| `bollinger_band` | `BollingerBandStrategy` | 布林带策略 | H1 |
| `mtf_ma_breakout` | `MultiTimeframeMABreakoutStrategy` | 多周期趋势 + MA 回调突破策略 | M15 + H1 |

### 策略参数说明

**ma_cross（双均线交叉）**
- `fast_period=5` — 快速均线周期
- `slow_period=20` — 慢速均线周期

**macd_cross（MACD 交叉）**
- `fast_period=12`, `slow_period=26`, `signal_period=9`

**rsi_ob_os（RSI 超买超卖）**
- `period=14`, `overbought=70.0`, `oversold=30.0`

**bollinger_band（布林带）**
- `period=20`, `std_dev=2.0`

**mtf_ma_breakout（多周期 MA 突破）**
- `ma_period=11` — MA 均线周期
- `risk_reward_ratio=2.0` — 盈亏比
- `min_pullback_bars=3` — 最小回调K线数
- `max_pullback_bars=5` — 最大回调K线数

| 路径 | 说明 | 认证 |
|------|------|------|

| `/api/v1/ws/trading` | 交易事件推送（订单、成交、持仓） | 必须 Token |
| `/api/v1/ws/strategy` | 策略信号推送 | 可选 Token |

#### 认证说明

- **无需认证**：`/register`、`/login`、`/password/reset`、健康检查探针（`/health/ready`、`/health/live`）、公开信号列表、排行榜查询
- **需要认证**：其余所有接口均需在请求头携带 `Authorization: Bearer <token>`
- **Token 有效期**：30 分钟，过期后需重新登录
- **限流**：每个 IP 每 60 秒最多 200 次请求

## WebSocket 实时推送

### 行情推送（`/api/v1/ws/market`）

连接后发送订阅消息即可接收实时行情数据，Token 可选（用于身份识别）。

```javascript
const ws = new WebSocket('ws://localhost:8000/api/v1/ws/market?token=<可选JWT>');

ws.onopen = () => {
  // 订阅频道
  ws.send(JSON.stringify({
    action: 'subscribe',
    channels: ['ticker/BTCUSDT', 'kline/BTCUSDT/1m', 'depth/BTCUSDT']
  }));
};

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  console.log(msg.type, msg.data);
};
```

**支持的频道：**

| 频道格式 | 说明 |
|----------|------|
| `ticker/{symbol}` | 实时 Ticker（最新价、涨跌幅、成交量） |
| `kline/{symbol}/{timeframe}` | K 线数据（实时更新） |
| `depth/{symbol}` | 市场深度（买卖盘） |

**客户端消息格式：**

```json
{"action": "subscribe",   "channels": ["ticker/BTCUSDT", "kline/BTCUSDT/1m"]}
{"action": "unsubscribe", "channels": ["ticker/BTCUSDT"]}
{"action": "ping"}
```

**服务端推送格式：**

```json
{"type": "ticker",  "channel": "ticker/BTCUSDT",   "data": {...}}
{"type": "kline",   "channel": "kline/BTCUSDT/1m",  "data": {...}}
{"type": "depth",   "channel": "depth/BTCUSDT",     "data": {...}}
{"type": "pong"}
{"type": "connected", "message": "行情 WebSocket 连接成功"}
```

### 交易事件推送（`/api/v1/ws/trading`）

需要 JWT Token 认证，连接后自动订阅当前用户的私有交易频道。

```javascript
const ws = new WebSocket('ws://localhost:8000/api/v1/ws/trading?token=<JWT Token>');

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  switch (msg.type) {
    case 'order_update': // 订单状态变化
    case 'trade':        // 成交通知
    case 'position':     // 持仓变化
    case 'account':      // 账户余额变化
  }
};
```

```javascript
const ws = new WebSocket('ws://localhost:8000/api/v1/ws/strategy?token=<可选JWT>');

ws.onopen = () => {
```

## 服务器部署

本节详细介绍如何将量化交易系统部署到 Linux 服务器上运行。

### 前置条件

- 一台 Linux 服务器（推荐 Ubuntu 20.04+ / CentOS 7+）
- 服务器已安装 Docker（>= 20.10）和 Docker Compose（>= 2.0）
- 服务器已安装 Git
- 服务器至少 2GB 内存（推荐 4GB+）
- 开放端口：8000（API）、3000（Grafana）、9090（Prometheus）

### 方式一：Docker Compose 一键部署（推荐）

这是最简单的方式，一条命令启动全部服务（包括数据库、缓存、消息队列、监控）。

#### 第一步：上传代码到服务器

```bash
# 在服务器上克隆代码
git clone <你的仓库地址> /opt/quant_trading_system
cd /opt/quant_trading_system
```

或者从本地上传：

```bash
# 在本地机器执行
rsync -avz --exclude='__pycache__' --exclude='.git' --exclude='*.pyc' \
  ./ user@your-server-ip:/opt/quant_trading_system/
```

#### 第二步：配置环境变量

```bash
cd /opt/quant_trading_system

# 编辑 .env 文件，填入生产配置
vi .env
```

生产环境关键配置：


## 常见问题

### Q1: 安装时报错 "ta-lib not found"

```bash
# macOS
brew install ta-lib
pip install ta-lib

# Ubuntu / Debian
sudo apt-get install -y ta-lib
```

### Q2: 运行 demo.py 报错

检查步骤：
1. 确认 Python 版本 >= 3.11：`python --version`
2. 确认已安装依赖：`pip install -e .`
3. 确认在项目根目录运行：`python examples/demo.py`

### Q3: 币安 API 连接失败

- 检查网络连接（部分地区可能需要代理）
- 确认 `.env` 中的 API Key 是否正确
- 使用 `--mock` 参数先用模拟数据测试

### Q4: Docker 容器启动失败

```bash
# 查看失败原因
docker compose logs <服务名>

# 检查端口冲突
sudo lsof -i :8000
sudo lsof -i :5432

# 检查磁盘空间
df -h
```

### Q5: 如何使用自己的数据？

```python
import pandas as pd
from quant_trading_system.models.market import BarArray

# 读取 CSV
df = pd.read_csv('your_data.csv')

# 转换为 BarArray 格式
bars = BarArray(
    symbol="BTC/USDT",
    exchange="custom",
    timeframe=TimeFrame.H1,

```

### Q6: 回测结果不理想怎么办？

1. 调整策略参数（均线周期、超买超卖阈值等）
2. 使用更长时间范围的数据回测
3. 尝试不同策略（如 `mtf_ma_breakout` 多周期策略）
4. 调低手续费率和滑点率查看参数敏感性
5. 开发自定义策略（参考 `src/quant_trading_system/strategies/examples.py`）

### Q7: API 调用返回 401 未授权

- 确认已通过 `POST /api/v1/c/user/login` 获取 Token
- 确认请求头格式正确：`Authorization: Bearer <token>`
- Token 有效期为 30 分钟，过期后需重新登录

### Q8: API 调用返回 503 交易系统未启动

- 行情、交易等接口依赖交易引擎，服务启动时会自动初始化
- 检查服务日志确认编排器是否启动成功：`docker compose logs -f quant-api`

### Q9: 策略 ID 格式是什么？

C 端策略 ID 格式为 `{username}__{snowflake_id}`，例如 `trader01__123456789`。
模拟交易策略 ID 格式为 `{username}__sim__{snowflake_id}`，例如 `trader01__sim__987654321`。
用户只能操作自己创建的策略（以自己的用户名开头）。

### Q10: WebSocket 连接断开后如何重连？

建议在客户端实现自动重连逻辑：

```javascript
function connectWS(url, onMessage) {
  const ws = new WebSocket(url);
  ws.onclose = () => setTimeout(() => connectWS(url, onMessage), 3000);
  ws.onmessage = (e) => onMessage(JSON.parse(e.data));
  return ws;
}
```

### Q11: 如何查看每笔交易的详细信息？

设置日志级别为 DEBUG，每笔交易成交时会记录方向、价格、数量、金额、手续费等信息：

```bash
LOG_LEVEL=DEBUG quant backtest --strategy ma_cross --symbol BTCUSDT --start "2024-01-01" --end "2024-03-31" --mock
```
