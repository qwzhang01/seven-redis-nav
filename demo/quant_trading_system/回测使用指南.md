# 回测使用指南

## 功能概述

本系统已集成完整的回测功能，支持：
- ✅ 从币安API获取真实历史数据
- ✅ 使用模拟数据进行测试
- ✅ 多种策略回测
- ✅ 详细的性能指标分析

## 快速开始

### 1. 查看可用策略

```bash
uv run quant list-strategies
```

输出示例：
```
Available strategies:
  - dual_ma          # 双均线策略
  - macd_cross       # MACD交叉策略
  - rsi_ob_os        # RSI超买超卖策略
  - bollinger_band   # 布林带策略
```

### 2. 运行回测（使用模拟数据）

```bash
uv run quant backtest \
  --strategy dual_ma \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-03-31" \
  --capital 100000 \
  --mock
```

### 3. 运行回测（使用币安真实数据）

```bash
uv run quant backtest \
  --strategy dual_ma \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-03-31" \
  --capital 100000
```

**注意**：使用真实数据需要网络连接到币安API。

## 命令参数说明

### 必需参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--strategy` | 策略名称 | `dual_ma` |
| `--symbol` | 交易对 | `BTCUSDT` |
| `--start` | 开始时间 | `2024-01-01` |
| `--end` | 结束时间 | `2024-03-31` |

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--timeframe` | 时间周期 | `1m` |
| `--capital` | 初始资金 | `1000000.0` |
| `--commission` | 手续费率 | `0.0004` (0.04%) |
| `--slippage` | 滑点率 | `0.0001` (0.01%) |
| `--mock` | 使用模拟数据 | `false` |

### 支持的时间周期

- `1m` - 1分钟
- `5m` - 5分钟
- `15m` - 15分钟
- `30m` - 30分钟
- `1h` - 1小时
- `4h` - 4小时
- `1d` - 1天
- `1w` - 1周

## 回测结果说明

回测完成后会显示以下指标：

### 基本信息
- **策略名称**：使用的策略
- **回测周期**：回测的天数

### 收益指标
- **初始资金**：起始资金
- **最终资金**：回测结束时的资金
- **总收益率**：整个回测期间的收益率
- **年化收益率**：按年计算的收益率

### 风险指标
- **最大回撤**：资金曲线的最大回撤幅度
- **波动率**：收益的标准差
- **夏普比率**：风险调整后的收益率（越高越好）
- **卡玛比率**：收益与最大回撤的比率（越高越好）

### 交易统计
- **总交易次数**：完成的交易数量
- **盈利次数**：盈利的交易数量
- **亏损次数**：亏损的交易数量
- **胜率**：盈利交易占比
- **盈亏比**：平均盈利与平均亏损的比率
- **平均盈利**：每笔盈利交易的平均收益
- **平均亏损**：每笔亏损交易的平均损失

### 费用统计
- **总手续费**：累计支付的手续费
- **总滑点**：累计的滑点成本

## 数据源说明

### 模拟数据（--mock）

- 使用几何布朗运动生成价格数据
- 包含随机波动和上涨趋势
- 适合快速测试和开发
- 不需要网络连接

### 币安真实数据

- 从币安REST API获取历史K线数据
- 真实的市场数据
- 需要网络连接
- 支持现货和合约市场

## 常见问题

### 1. 策略没有产生交易

可能原因：
- 数据量不足（增加回测时间范围）
- 策略参数不合适
- 市场条件不满足策略信号

解决方法：
- 增加回测时间范围
- 调整策略参数
- 尝试其他策略

### 2. 币安API连接失败

可能原因：
- 网络问题
- SSL证书问题
- API限流

解决方法：
- 检查网络连接
- 使用`--mock`参数测试
- 减少请求频率

### 3. 如何添加自定义策略

参考 `src/quant_trading_system/strategies/examples.py` 中的示例：

```python
from quant_trading_system.services.strategy.base import Strategy, register_strategy

@register_strategy("my_strategy")
class MyStrategy(Strategy):
    """我的自定义策略"""
    
    def on_bar(self, bar):
        # 实现策略逻辑
        pass
```

## 技术架构

### 核心组件

1. **BinanceAPI** (`services/market/binance_api.py`)
   - 币安REST API客户端
   - 获取历史K线数据
   - 支持重试和错误处理

2. **MockData** (`services/market/mock_data.py`)
   - 模拟数据生成器
   - 使用几何布朗运动
   - 可配置波动率和趋势

3. **BacktestEngine** (`services/backtest/backtest_engine.py`)
   - 事件驱动回测引擎
   - 支持多品种回测
   - 完整的性能指标计算

4. **CLI** (`cli.py`)
   - 命令行接口
   - 参数解析和验证
   - 结果格式化输出
