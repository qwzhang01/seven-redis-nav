# 回测使用指南

## 功能概述

系统集成了完整的策略回测引擎，支持：

- 从币安 API 获取真实历史 K 线数据
- 使用模拟数据（几何布朗运动）进行快速测试
- 多品种、多时间周期回测
- 交易成本模拟（手续费 + 滑点）
- 回测进度实时日志（进度百分比、处理速度、ETA）
- 完整的绩效指标分析（收益、风险、交易统计）
- 通过 API 异步提交回测任务，支持查询结果、权益曲线、交易记录

## CLI 命令行回测

### 1. 查看可用策略

```bash
quant list-strategies
```

输出示例：

```
Available strategies:
  - ma_cross            # 双均线交叉策略
  - macd_cross          # MACD交叉策略
  - rsi_ob_os           # RSI超买超卖策略
  - bollinger_band      # 布林带策略
  - mtf_ma_breakout     # 多周期趋势+MA回调突破策略
```

### 2. 运行回测（使用模拟数据）

```bash
quant backtest \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000 \
  --mock
```

### 3. 运行回测（使用币安真实数据）

```bash
quant backtest \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000
```

> 使用真实数据需要网络连接到币安 API。

### 4. 运行多周期策略回测

多周期策略（如 `mtf_ma_breakout`）会自动获取多个时间周期的数据：

```bash
quant backtest \
  --strategy mtf_ma_breakout \
  --symbol BTCUSDT \
  --timeframe 15m \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000 \
  --mock
```

系统会自动检测策略定义的时间周期（如 M15 + H1），分别获取对应数据。

### 5. 调整手续费和滑点

```bash
quant backtest \
  --strategy ma_cross \
  --symbol BTCUSDT \
  --timeframe 1h \
  --start "2024-01-01" \
  --end "2024-06-30" \
  --capital 100000 \
  --commission 0.001 \
  --slippage 0.0005 \
  --mock
```

## API 接口回测

除 CLI 外，系统还提供 RESTful API 接口，支持异步提交回测任务并查询结果。

### 1. 提交回测任务

```bash
curl -X POST http://localhost:8000/api/v1/c/backtest/run \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "strategy_type": "ma_cross",
    "symbol": "BTCUSDT",
    "timeframe": "1h",
    "start_time": "2024-01-01T00:00:00",
    "end_time": "2024-06-30T00:00:00",
    "initial_capital": 100000.0,
    "commission_rate": 0.0004,
    "slippage_rate": 0.0001,
    "params": {
      "fast_period": 5,
      "slow_period": 20
    }
  }'
```

返回：

```json
{
  "success": true,
  "backtest_id": "backtest_123456789",
  "message": "Backtest started successfully"
}
```

### 2. 查询回测状态与结果

```bash
# 查询回测结果（running/completed/failed）
curl http://localhost:8000/api/v1/c/backtest/backtest_123456789 \
  -H "Authorization: Bearer <token>"
```

返回（完成时）：

```json
{
  "backtest_id": "backtest_123456789",
  "status": "completed",
  "result": {
    "strategy_name": "ma_cross",
    "total_return": 0.105,
    "annual_return": 0.2235,
    "max_drawdown": 0.0523,
    "sharpe_ratio": 1.43,
    "total_trades": 45,
    "win_rate": 0.6222
  }
}
```

### 3. 获取权益曲线

```bash
curl http://localhost:8000/api/v1/c/backtest/backtest_123456789/equity \
  -H "Authorization: Bearer <token>"
```

### 4. 获取交易记录

```bash
curl "http://localhost:8000/api/v1/c/backtest/backtest_123456789/trades?limit=100" \
  -H "Authorization: Bearer <token>"
```

### 5. 查看历史回测列表

```bash
curl "http://localhost:8000/api/v1/c/backtest/list?limit=20" \
  -H "Authorization: Bearer <token>"
```

### 6. 删除回测记录

```bash
curl -X DELETE http://localhost:8000/api/v1/c/backtest/backtest_123456789 \
  -H "Authorization: Bearer <token>"
```

## 回测结果说明

回测完成后会显示以下指标：

### 基本信息

- **策略名称** — 运行的策略
- **回测周期** — 回测覆盖的天数

### 收益指标

- **初始资金** — 回测起始资金
- **最终资金** — 回测结束时的账户权益
- **总收益率** — 整个回测期间的收益率
- **年化收益率** — 按年折算的收益率

### 风险指标

- **最大回撤** — 权益曲线从峰值到谷底的最大跌幅
- **波动率** — 收益率的年化标准差
- **夏普比率** — 风险调整后的收益率（越高越好，>1 为优）
- **卡玛比率** — 年化收益率 / 最大回撤（越高越好）

### 交易统计

- **总交易次数** — 完成的交易笔数
- **盈利次数** — 盈利的交易笔数
- **亏损次数** — 亏损的交易笔数
- **胜率** — 盈利交易占比
- **盈亏比** — 总盈利 / 总亏损
- **平均盈利** — 每笔盈利交易的平均收益
- **平均亏损** — 每笔亏损交易的平均损失

### 费用统计

- **总手续费** — 累计支付的手续费
- **总滑点** — 累计的滑点成本

## 内置策略说明

### ma_cross — 双均线交叉策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `fast_period` | 5 | 快速均线周期 |
| `slow_period` | 20 | 慢速均线周期 |

买入：快线上穿慢线；卖出：快线下穿慢线。

### macd_cross — MACD 交叉策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `fast_period` | 12 | MACD 快线周期 |
| `slow_period` | 26 | MACD 慢线周期 |
| `signal_period` | 9 | 信号线周期 |

买入：MACD 线上穿信号线；卖出：MACD 线下穿信号线。

### rsi_ob_os — RSI 超买超卖策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `period` | 14 | RSI 计算周期 |
| `overbought` | 70.0 | 超买阈值 |
| `oversold` | 30.0 | 超卖阈值 |

买入：RSI 从下方穿越超卖线；卖出：RSI 从上方穿越超买线。

### bollinger_band — 布林带策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `period` | 20 | 布林带周期 |
| `std_dev` | 2.0 | 标准差倍数 |

买入：价格触及下轨；卖出：价格触及上轨。

### mtf_ma_breakout — 多周期趋势 + MA 回调突破策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ma_period` | 11 | MA 均线周期 |
| `risk_reward_ratio` | 2.0 | 盈亏比 |
| `min_pullback_bars` | 3 | 最小回调 K 线数 |
| `max_pullback_bars` | 5 | 最大回调 K 线数 |

**策略逻辑：**
1. 大周期（H1）确认趋势方向：阳线且开盘价 > MA(11) 为多头
2. 小周期（M15）寻找回调突破入场：收盘价从 MA 下方突破到上方，且突破前存在 3-5 根回调 K 线
3. 止损设在回调结构最低价，止盈 = 入场价 + 风险 × 盈亏比

## 数据源说明

### 模拟数据（--mock）

- 使用几何布朗运动生成价格序列
- 内置先涨后跌的趋势 + 正弦波动，确保产生交叉信号
- 多周期模式下先生成最小周期数据，再聚合为更高周期（OHLCV 正确聚合）
- 适合快速测试和开发，不需要网络连接

### 币安真实数据

- 通过币安 REST API 获取历史 K 线数据
- 支持现货和合约市场
- 需要网络连接
- 数据量较大时获取时间较长

### 数据库历史数据（API 回测）

通过 API 提交的回测任务使用 `BacktestEngine.run_from_database()` 方法，从 TimescaleDB 中读取已存储的历史 K 线数据。需要先通过 Admin 端订阅管理接口配置并启动行情订阅，将历史数据同步到数据库。

## 回测引擎技术细节

### 多周期处理机制

- 以所有周期中最小的作为主驱动周期
- 每根主周期 K 线到来时，对高周期数据按时间戳切片（`<= 当前时间`），严格避免未来函数
- 策略通过 `context.bars[symbol][timeframe]` 访问各周期数据

### 预热期

- 固定 100 根 K 线作为预热期，确保指标计算有足够的历史数据
- 预热期内不执行策略逻辑

### 交易成本模拟

- **手续费**：`max(数量 × 价格 × 费率, 最低手续费)`
- **滑点**：买入价 = 收盘价 + 滑点，卖出价 = 收盘价 - 滑点
- 滑点 = 价格 × 滑点率 + 固定滑点

### API 回测异步机制

通过 API 提交的回测任务以异步方式执行（`asyncio.create_task`），提交后立即返回 `backtest_id`。客户端可轮询 `GET /api/v1/c/backtest/{backtest_id}` 查询状态：

- `running`：回测正在执行中
- `completed`：回测完成，可查询结果
- `failed`：回测失败，返回错误信息

## 自定义策略开发

参考 `src/quant_trading_system/strategies/examples.py` 创建新策略：

```python
from quant_trading_system.services.strategy.base import Strategy, register_strategy
from quant_trading_system.services.strategy.signal import Signal, SignalType
from quant_trading_system.models.market import Bar, TimeFrame

@register_strategy("my_strategy")
class MyStrategy(Strategy):
    """自定义策略"""

    name = "my_strategy"
    description = "我的自定义策略"
    timeframes = [TimeFrame.H1]

    # 参数定义（用于 API 展示和校验）
    params_def = {
        "fast_period": {"type": int, "default": 10, "description": "快速均线周期"},
        "slow_period": {"type": int, "default": 30, "description": "慢速均线周期"},
    }

    def on_init(self):
        """策略初始化"""
        self.params.setdefault("fast_period", 10)
        self.params.setdefault("slow_period", 30)

    def on_bar(self, bar: Bar):
        """每根K线到来时调用"""
        bars = self.context.bars[bar.symbol][bar.timeframe]
        ind = self.context.indicator_engine

        fast_ma = ind.sma(bars.close, self.params["fast_period"])
        slow_ma = ind.sma(bars.close, self.params["slow_period"])

        if fast_ma[-1] > slow_ma[-1] and fast_ma[-2] <= slow_ma[-2]:
            return Signal(
                symbol=bar.symbol,
                signal_type=SignalType.LONG_ENTRY,
                price=bar.close,
                strategy_id=self.strategy_id,
            )

        if fast_ma[-1] < slow_ma[-1] and fast_ma[-2] >= slow_ma[-2]:
            return Signal(
                symbol=bar.symbol,
                signal_type=SignalType.LONG_EXIT,
                price=bar.close,
                strategy_id=self.strategy_id,
            )

        return None
```

将策略文件放入 `src/quant_trading_system/strategies/` 目录，并在 `__init__.py` 中导入即可自动注册。

注册后可通过以下方式使用：

```bash
# CLI 回测
quant backtest --strategy my_strategy --symbol BTCUSDT --start "2024-01-01" --end "2024-06-30" --mock

# API 查询策略类型
curl http://localhost:8000/api/v1/c/strategy/types
# 返回中会包含 my_strategy 的参数定义和时间周期信息
```

## 常见问题

### 1. 策略没有产生交易

- 数据量不足：增加回测时间范围
- 预热期消耗：系统固定 100 根 K 线预热，确保数据量 > 100
- 策略参数不合适：尝试调整参数或换用其他策略

### 2. 币安 API 连接失败

- 检查网络连接
- 使用 `--mock` 参数先用模拟数据验证
- 减少请求频率，避免 API 限流

### 3. 回测速度慢

- 减少回测时间范围
- 使用较大的时间周期（如 `1h` 代替 `1m`）
- 回测进度日志中的 `speed` 字段可以帮助判断性能瓶颈

### 4. 如何查看每笔交易的详细信息？

设置日志级别为 DEBUG，每笔交易成交时会记录方向、价格、数量、金额、手续费等信息：

```bash
LOG_LEVEL=DEBUG quant backtest --strategy ma_cross --symbol BTCUSDT --start "2024-01-01" --end "2024-03-31" --mock
```

### 5. API 回测返回 running 状态很久没有完成？

- 检查数据库中是否有对应时间段的 K 线数据
- 查看服务日志确认回测任务是否报错：`docker compose logs -f quant-api`
- 如果数据库中没有数据，请先通过 Admin 端订阅管理接口配置行情订阅，或改用 CLI 命令行回测（支持 `--mock` 模拟数据）

### 6. 如何对比多个策略的回测结果？

可以分别运行多个回测任务，然后通过 `GET /api/v1/c/backtest/list` 查看历史回测列表，对比各策略的收益率、夏普比率、最大回撤等指标。

### 7. 回测结果中的年化收益率是如何计算的？

年化收益率 = `(1 + 总收益率) ^ (365 / 回测天数) - 1`

例如：回测 181 天，总收益率 10.5%，则年化收益率 ≈ `(1.105)^(365/181) - 1 ≈ 22.35%`
