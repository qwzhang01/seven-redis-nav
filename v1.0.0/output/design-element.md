## 第4章：设计约束与建议

### 4.1 设计风格约束

| 约束类型 | 约束内容 | 建议处理方式 |
|---------|---------|-------------|
| 整体风格 | 金融量化交易风格，参考TradingView | 深色主题为主，数据密集展示 |
| 配色方案 | 涨跌使用绿/红色（国际惯例：绿涨红跌） | 绿色#00A870，红色#E34D59 |
| 数据更新 | 行情数据需实时更新（WebSocket） | 价格变动时使用闪烁动效 |
| 图表组件 | K线图需使用专业图表库 | 使用Lightweight Charts或TradingView |
| 响应性能 | 价格推送延迟 < 100ms | WebSocket实时推送 |

### 4.2 技术约束

| 约束类型 | 约束内容 | 建议处理方式 |
|---------|---------|-------------|
| 组件库 | 使用TDesign组件库 | 通过CDN引入TDesign Web Components |
| 原型文件 | 单HTML文件，可直接浏览器打开 | 内联CSS和JS |
| K线图表 | 需支持多种时间周期和指标 | 使用Lightweight Charts库 |
| 代码编辑器 | 需支持Python语法高亮 | 使用Monaco Editor CDN |

### 4.3 待确认项

| 待确认项 | 推断内容 | 确认建议 |
|---------|---------|---------|
| 涨跌色彩 | 国际惯例（绿涨红跌） | 确认是否需要切换为国内惯例（红涨绿跌） |
| 交易所数量 | MVP支持Binance/OKX/Bybit | 确认具体支持的交易所列表 |
| 合约交易 | MVP不包含合约交易 | 确认是否需要在原型中预留入口 |
