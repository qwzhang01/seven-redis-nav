## 第1部分：Epic 概述分析

| 分析维度 | 内容 |
|---------|------|
| Epic 名称 | 加密货币量化交易系统 |
| Epic 类型 | **业务 Epic** - 直接交付个人投资者的量化交易价值 |
| 业务价值 | 构建面向个人投资者的一站式加密货币量化交易平台，集行情、交易、策略、社区于一体 |
| 核心目标 | 降低个人投资者量化交易门槛，提供策略研发、回测、实盘一体化服务 |
| 成功标准 | P0阶段用户注册量≥1000；系统可用性≥99.5%；策略订阅转化率≥10%；行情延迟<500ms |
| 复杂度评估 | **高** - 涉及10+用户角色、多交易所接入、实时行情、策略引擎、风控体系、多国合规 |

| 国际化 | 中 | 中英文双语支持 |

### Feature 验收标准

#### P0 核心 Feature

- **F001 实时行情与K线图表**:
  - 支持BTC/USDT等20+主流交易对
  - K线周期支持：1m/5m/15m/30m/1h/4h/1d/1w/1M
  - 技术指标支持≥20种（MA/MACD/RSI/布林带等）
  - 画线工具支持≥10种
  - 行情延迟<500ms，图表渲染≥30fps

- **F002 手动交易下单（现货）**:
  - 支持市价单、限价单、止盈止损单
  - 下单响应时间<1秒
  - 订单提交成功率≥99%
  - 支持一键买入/卖出预设比例

- **F002B 合约交易基础（⭐新增）**:
  - 支持USDT永续合约
  - 支持做多/做空双向交易
  - 支持1x-20x杠杆倍数选择
  - 支持逐仓保证金模式
  - 显示预估强平价格
  - 显示保证金率和未实现盈亏
  - 合约持仓管理（查看、平仓、加仓、减仓）
  - 合约止盈止损单
  - 首次合约交易风险确认
  - 合约下单响应<1秒

- **F003 订单管理系统**:
  - 当前委托列表实时更新（现货+合约）
  - 支持单笔撤单、批量撤单、一键撤单
  - 订单状态追踪（8种状态）
  - 历史订单查询和导出
  - 合约订单独立管理

- **F004 模拟交易系统（⭐新增）**:
  - 使用真实实时行情数据
  - 虚拟账户初始资金10,000 USDT（可配置）
  - 支持市价单/限价单/止盈止损单模拟
  - 订单撮合基于真实行情价格
  - 模拟手续费按真实费率（0.1%）
  - 模拟下单响应<500ms
  - 模拟/实盘模式一键切换
  - 模拟模式明显视觉标识
  - 模拟数据与实盘数据完全隔离
  - 虚拟账户可随时重置
  - 支持模拟交易统计报告（收益率、胜率、最大回撤等）

- **F005 策略广场与订阅**:
  - P0阶段系统策略≥10个
  - 策略分类展示（趋势跟踪/网格/套利等）
  - 策略详情页（描述、历史绩效、参数说明）
  - 策略订阅转化率≥10%

- **F006 策略回测引擎**:
  - 回测数据支持≥1年历史数据
  - 绩效指标完整（收益率、最大回撤、夏普比率、胜率、盈亏比）
  - 回测执行时间<30秒（1年数据）
  - 考虑手续费和滑点模型

- **F007 策略实盘运行**:
  - 策略信号到下单延迟<2秒
  - 订单执行成功率≥99%
  - 支持手动暂停/恢复策略
  - 实时运行状态监控

- **F007B 策略模拟运行（⭐增强）**:
  - 策略在模拟环境运行
  - 使用虚拟账户资金
  - 策略信号实时生成
  - 模拟订单自动执行
  - 支持多策略并行模拟运行
  - 策略参数可配置
  - 策略信号日志记录
  - 策略模拟绩效实时追踪
  - 模拟与回测结果对比
  - 策略运行状态监控（启动/停止/暂停/恢复）
  - 策略异常告警

- **F008 风控管理系统**:
  - 止损止盈机制（策略级/账户级）
  - 仓位控制（单策略/总仓位/单笔限额）
  - 紧急控制（一键暂停/一键平仓/熔断）
  - 异常检测（API异常/价格异常/资金异常）
  - 风控检查延迟<100ms

- **F009 用户账户系统**:
  - 支持邮箱/手机号注册登录
  - 支持两步验证（2FA）
  - 注册完成率≥70%
  - 登录成功率≥99%

- **F010 交易所配置管理**:
  - 支持Binance/OKX/Hyperliquid
  - API Key AES-256加密存储
  - API Key验证时间<5秒
  - 配置成功率≥95%

- **F011 基础通知系统**:
  - 站内通知延迟<5秒
  - 邮件送达率≥99%
  - 支持通知偏好配置

- **F012 基础客服支持**:
  - FAQ知识库覆盖常见问题
  - 工单首次响应<24小时
  - 官方Telegram/Discord群入口

## 第4部分：拆解策略

**推荐拆解维度**: **按功能模块 + 按用户场景** 混合拆解

**理由**: 
- 加密货币量化交易系统功能模块边界清晰（行情、交易、策略、用户、风控等）
- 各模块可并行开发，降低依赖风险
- 同时按用户场景拆分，确保每个模块端到端可交付、用户体验完整

## 第5部分：MVP 定义

| 维度 | 内容 |
|------|------|
| **MVP 范围** | F001 + F002 + F002B + F003 + F004 + F005 + F006 + F007 + F007B + F008 + F009 + F010（共93个Story） |
| **支持交易所** | Binance（首选）、OKX（次选）、Hyperliquid（P1扩展） |

| **核心假设** | 1. 交易所API稳定可用；2. 用户愿意配置API Key；3. 系统策略有正向收益；4. 模拟交易能有效降低新手门槛；5. 合约交易需求旺盛 |

| **预计交付** | 6个迭代（3个月） |

| **行情数据延迟** | 中 | 策略执行效果下降 | 多数据源备份，延迟监控告警 |
| **策略引擎性能** | 中 | 回测慢，实盘延迟高 | 性能优化，分布式处理 |
| **风控规则失效** | 中 | 用户大额亏损 | 多层风控，熔断机制，人工监控 |
| **时序数据库性能** | 中 | 历史数据查询慢 | 数据分层，冷热分离，索引优化 |
| **用户API Key配置失败** | 中 | 无法使用实盘功能 | 详细指引，权限检测，客服支持 |
| **合规风险** | 中 | 法律问题 | 与法务确认，用户协议完善，风险提示 |
| **市场极端波动** | 中 | 系统压力大，策略异常 | 熔断机制，降级方案，弹性扩容 |
