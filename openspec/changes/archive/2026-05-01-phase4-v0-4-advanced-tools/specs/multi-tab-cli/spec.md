## ADDED Requirements

### Requirement: CLI 多标签页容器
系统 SHALL 将 CLI 工作区改造为多标签页架构，在工作区顶部显示标签页栏（CliTabBar），每个标签页独立持有 Redis 连接、命令历史和输出内容。

#### Scenario: 初始状态
- **WHEN** 用户首次切换到 CLI Tab
- **THEN** 显示一个默认标签页（Tab 1），标签页栏显示"终端 1"，行为与 Phase 1 单 Tab CLI 完全一致

#### Scenario: 新建标签页
- **WHEN** 用户按 ⌘T 或点击标签页栏右侧的 "+" 按钮
- **THEN** 新建一个 CLI 标签页，标签名为"终端 N"（N 自增），自动切换到新标签页，新标签页建立独立的 Redis 连接

#### Scenario: 标签页数量限制
- **WHEN** 用户尝试新建第 9 个标签页（超过最大限制 8 个）
- **THEN** "+" 按钮变为禁用状态，鼠标悬停显示提示"最多支持 8 个终端标签页"

### Requirement: CLI 标签页切换
系统 SHALL 支持通过点击标签页标签或键盘快捷键切换当前激活的 CLI 标签页。

#### Scenario: 点击切换
- **WHEN** 用户点击标签页栏中的某个标签
- **THEN** 工作区切换到该标签页的输出内容和输入框，输入框自动聚焦

#### Scenario: 键盘切换
- **WHEN** 用户按 ⌘1 ~ ⌘8（在 CLI Tab 内）
- **THEN** 切换到对应序号的 CLI 标签页（如 ⌘1 切换到第 1 个标签页）

### Requirement: CLI 标签页关闭
系统 SHALL 支持关闭 CLI 标签页，关闭时释放对应的 Redis 连接。

#### Scenario: 关闭当前标签页
- **WHEN** 用户按 ⌘W 或点击标签页上的 "×" 按钮
- **THEN** 关闭当前标签页，释放其 Redis 连接，切换到相邻标签页；若只剩一个标签页则不允许关闭（"×" 按钮隐藏）

#### Scenario: 关闭有输出内容的标签页
- **WHEN** 用户关闭一个有命令执行历史的标签页
- **THEN** 直接关闭，不弹出确认对话框（历史内容不持久化）

### Requirement: CLI 标签页独立会话
系统 SHALL 为每个 CLI 标签页维护独立的 Redis 连接（`CliSession`），互不干扰。

#### Scenario: 独立命令历史
- **WHEN** 用户在标签页 1 执行若干命令后切换到标签页 2
- **THEN** 标签页 2 的 ↑↓ 历史导航只显示标签页 2 自己的命令历史，不包含标签页 1 的历史

#### Scenario: 独立 DB 选择
- **WHEN** 用户在标签页 1 执行 `SELECT 3` 切换到 DB 3
- **THEN** 标签页 2 仍在 DB 0（或其自己选择的 DB），两个标签页的 DB 状态互相独立

#### Scenario: 空闲标签页超时
- **WHEN** 某个 CLI 标签页超过 30 分钟无命令执行
- **THEN** 后端自动释放该标签页的 Redis 连接，标签页标签显示"(已断开)"，用户下次输入命令时自动重连

### Requirement: CLI 标签页重命名
系统 SHALL 支持双击标签页标签进行重命名。

#### Scenario: 重命名标签页
- **WHEN** 用户双击标签页标签
- **THEN** 标签名变为可编辑输入框，回车或失焦确认新名称，Escape 取消编辑
